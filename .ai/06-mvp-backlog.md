# 06 - MVP Backlog (Prioritized)

## P0 - Required for MVP

1. As a first-time operator, I can create the initial admin account during setup.
2. As an admin, I can configure whether user self-registration is `OPEN` or `ADMIN_ONLY`.
3. As a user, I can sign in with password or passkey.
4. As an admin, I can manage fine-grained permissions using CASL abilities.
5. As an admin, I can assign permissions via role policies and user-level overrides.
6. As a staff user, I can manage locations and freely nested containers.
7. As a staff user, I can create and edit items.
8. As a staff user, I can view stock by container.
9. As a staff user, I can create `IN/OUT/TRANSFER/ADJUSTMENT` movements.
10. As a staff user, I must enter a reason for `OUT` and `ADJUSTMENT` movements.
11. As a staff user, I can use a persistent mobile navbar scan button from core screens.
12. As a staff user, I can scan QR codes and jump directly to resolved containers and items.
13. As a staff user, I can stock out an item directly from the scan result screen.
14. As a staff user, I can define item size metadata (e.g. package size, servings).
15. As a staff user, I can manage an internal image library for reusable inventory photos.
16. As a staff user, I can select images from the internal image library during stock-in.
17. As a staff user, I can export QR labels as PDF files using the `A4_GRID` template.
18. As a staff user, I can attach images to containers/items.
19. As a staff user, I can attach images to storage locations and containers.
20. As a staff user, I can choose icons for locations, containers, categories, and items.
21. As a staff user, I can select icons from Tabler and Lucide sets.
22. As a user, I can view full movement history with actor context.
23. As an operator, I have audit logs for critical actions.
24. As a user, I can use the app as a mobile PWA.
25. As a user, I can continue core reads and queue writes while offline.
26. As a user, I can switch UI language between German and English.
27. As a user, my selected language is saved to my account profile.
28. As a user, my selected theme is saved locally in my browser.

## P1 - Important after MVP

1. Offline conflict resolution hints for queued writes.
2. CSV import for master data onboarding.
3. Expanded dashboard KPIs.
4. Additional label templates beyond `A4_GRID`.

## P2 - Later Expansion

1. Multi-tenant support.
2. External integrations (ERP/procurement).
3. Advanced inventory campaigns with review/approval flows.
